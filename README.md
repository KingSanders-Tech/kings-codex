# Kings Codex

A personal reference library of commands, scripts, and workflows I've collected and built while transitioning into DevSecOps and AppSec engineering. Every entry here is something I've actually run, tested, or built. Nothing is theoretical filler.

This repo doubles as my second brain. It lives locally as an [Obsidian](https://obsidian.md) vault and remotely on GitHub so I have it on any machine I touch.

## Why this exists

Memory is a bad place to store working code. Half the time I needed a command, I was hunting through old terminal history, screenshots, or Slack DMs to myself. This fixes that.

It also gives me a single place to capture the security audits, deployment patterns, and agent workflows I'm building inside the [KINGS_OS](./kings-os/) ecosystem and [Hawkly](./hawkly/).

## How it's organized

| Folder | What's in it |
|---|---|
| [`security/`](./security/) | Manual audits, hardening procedures, recon and triage commands |
| [`devops/`](./devops/) | Docker, deployment, CI/CD, infrastructure snippets |
| [`development/`](./development/) | Language and tool cheatsheets, git recovery, debugging |
| [`kings-os/`](./kings-os/) | Agent commands and workflows for my local AI mesh |
| [`hawkly/`](./hawkly/) | Deployment notes and operational runbooks for Hawkly |
| [`_templates/`](./_templates/) | Markdown templates I copy when adding new entries |

## How I use it

- **Local**: opened as an Obsidian vault for daily reference and note linking
- **Sync**: pushed to GitHub from the terminal so it's available anywhere
- **Diff**: every commit captures what I learned that day
